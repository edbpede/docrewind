# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

DocRewind is a **local-first MV3 browser extension** (Chrome + Firefox from one codebase) that
reconstructs and replays the revision history of a Google Doc entirely on-device — no backend,
account, or telemetry. Stack: SolidJS + WXT (Vite) + Bun + UnoCSS (presetWind4) + Biome.

## Commands

**Bun-only — never use `npm`/`npx`/`pnpm`; use `bun` / `bunx`.** Run `bun run postinstall`
(`wxt prepare`, generates `.wxt/` types) once after install, before `compile` or `build`.

| Task | Command |
|---|---|
| Install | `bun install --frozen-lockfile` then `bun run postinstall` |
| Dev (HMR) | `bun run dev` · `bun run dev:firefox` |
| Build | `bun run build` (→ `.output/chrome-mv3`) · `bun run build:firefox` |
| Typecheck | `bun run compile` (`tsc --noEmit`) |
| Lint/format (fix) | `bun run check` (`biome check --write .`); CI uses `bunx biome ci .` (no writes) |
| Pure-logic tests | `bun run test:logic` (Bun, `bun:test`) |
| Component/storage tests | `bun run test:run` (Vitest, CI) · `bun run test` (watch) |
| Coverage gate | `bun run test:coverage` (decoder+reconstruction, ≥85% per-file) |
| E2E | `bun run test:e2e` (Playwright; **build first**, runs against `.output/chrome-mv3`) |
| Pre-commit hooks | `bun run hooks` (all files) · `bun run hooks:install` |

Single test (Bun): `bun test ./lib/theme.test.ts` · by name: `bun test ./lib/decoder -t "<name>"`
Single test (Vitest): `bunx vitest run test/smoke.test.tsx` · by name: add `-t "<name>"`

## Test-runner split (this trips people up)

Tests are partitioned by directory; the two runners are mutually exclusive (`vitest.config.ts`
*excludes* the Bun dirs, `test:logic` *only* runs them). Put a new test in the right tier:

| Test subject | Runner | Lives in |
|---|:---:|:---:|
| Pure logic (no DOM, imports `bun:test`) | `bun test` | `lib/{decoder,reconstruction,timeline,domain,protocol,retrieval,worker,docs-url,replay,i18n,identity}`, `lib/theme.test.ts` |
| Solid components, storage, messaging (imports `vitest`) | Vitest | `test/**`, `lib/*.test.ts` (e.g. `lib/db.test.ts`) |
| Assembled extension | Playwright | `e2e/**` |

When adding a Bun-only `lib/` subtree, also add it to the `exclude` list in `vitest.config.ts`
and to the `test:logic` path list in `package.json`.

## Architecture and boundaries

Data flows one direction across context boundaries; the typed contract is `lib/messaging.ts`
(`ProtocolMap`, via `@webext-core/messaging`):

1. **Content script** `entrypoints/docs.content.tsx` detects a doc, sends `activateReplay` /
   `startRetrieval`. Never auto-loads history.
2. **Background** `entrypoints/background.ts` owns retrieval: the only live `fetch` (cookie-auth,
   `credentials: "include"`) and the idb checkpoint store, wired into the pure orchestrator
   `lib/retrieval/orchestrator.ts` (resumable discover→fetch→checkpoint loop, all deps injected).
3. Raw chunks persist to IndexedDB via `lib/db.ts` (interface `RevisionStore` in `lib/store.ts`;
   `lib/db.memory.ts` is the in-memory store for tests).
4. **Pipeline** `lib/worker/pipeline.ts` decodes (`lib/decoder`) → reconstructs
   (`lib/reconstruction`) → derives timeline (`lib/timeline`), then publishes one atomic
   `ReplayPublication`.
5. **Replay page** `entrypoints/replay/App.tsx` reads via `lib/replay/load.ts` (`loadReplayData`)
   and renders runs via `lib/reconstruction/render.ts` (`segmentsAt`) into
   `components/DocumentViewport.tsx` and `components/Timeline.tsx`.

Supporting layers: `lib/domain` (branded `DocId`/`RevisionId`, model types), `lib/protocol`
(Google Docs wire framing/endpoints/discovery), `lib/identity` (author/collaborator resolution),
`lib/settings.ts` (storage leases + budgets). Errors use the `Result`/`RetrievalError` type from
`lib/retrieval/errors.ts`, never thrown for expected failures.

## Repository-specific rules

- **Pure core must stay browser-free.** `lib/{decoder,reconstruction,timeline,domain,protocol,fixtures}`
  must not import `#imports`, `browser.*`, or `wxt`. Additionally `lib/{retrieval,worker,docs-url}`
  must not call `fetch(`, `new Worker`, or `globalThis`. The live `fetch` adapter belongs only in
  `entrypoints/background.ts`. Enforced by `scripts/check-pure-core.sh` (prek + CI).
- **Never hand-author `manifest.json`.** It is generated by WXT from `wxt.config.ts`. The privacy
  invariant lives there: `permissions: ["storage"]`, `host_permissions: ["*://docs.google.com/*"]`
  only. Enforced by `scripts/verify-manifest.sh` and `scripts/check-no-foreign-hosts.sh` (no
  non-Google network target may appear in source).
- **Background API placement.** All `browser.*` / `fetch` usage must be inside `defineBackground`,
  never at module top level (top level runs in WXT's Node build context and throws).
- **SolidJS, not React.** Components run **once** — read signals inside JSX. Do not destructure
  props, use `useState`/`useEffect`, or derive state in `createEffect`. Use `<For>`/`<Show>`, not
  `.map()`/ternaries, for lists/conditionals.
- **Coverage floor is per-file.** Any new file in `lib/decoder` or `lib/reconstruction` must clear
  85% line+function coverage from its first commit (Bun has no aggregate grace). To exempt a thin
  adapter, add its path glob to `coveragePathIgnorePatterns` in `bunfig.toml` with a justification
  — do not lower the global threshold.
- **Committed generated artifacts:** `public/icon/{16,32,48,96,128}.png` are committed for build
  determinism. Regenerate only when the SVG art changes: `./scripts/generate-icons.sh`.
- TypeScript is strict + `noUncheckedIndexedAccess` + `exactOptionalPropertyTypes`; Biome forbids
  `any` (`noExplicitAny: error`). The `@/*` path alias maps to the repo root.

## Reconstruction invariant (read before touching render/snapshot)

`segmentsAt(model)` is **single-arg by design**: the model is *already* time-traveled by
`modelAtRevisionIndex(i)`; there is deliberately no `t`/index parameter (mixing the applied-count
scale with the wire `RevisionId` scale corrupts the view). Segment visibility mirrors
`currentText` exactly — accepted-deleted elements are tombstoned out. See the header docblock in
`lib/reconstruction/render.ts`.
