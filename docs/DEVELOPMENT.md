# Developer guide — reproduce a build from a clean machine

This guide takes a contributor from a fresh clone to a working dev build, a full
test run, and packaged extensions. It is the from-clean-machine reproduction
reference for PRD §17 ("contributors can reproduce a build and run tests from
documented steps on a clean machine"). For the release runbook and build
provenance, see [`RELEASE.md`](RELEASE.md); for the contribution rules (commits,
hooks, DCO, SPDX), see [`../CONTRIBUTING.md`](../CONTRIBUTING.md).

## Prerequisites

| Tool | Version | Why |
|------|---------|-----|
| [Bun](https://bun.sh) | **1.3.14** (exact) | Package manager, script runner, pure-logic test runtime. This is the only required toolchain — see [`STACK.md`](STACK.md). |
| [`prek`](https://prek.j178.dev) | latest | Git-hook framework (Rust). Needed only if you intend to commit. |
| Git | any recent | Clone + DCO sign-off. |

**No Node.js install is required** — Bun runs the TypeScript build and scripts.
Other tool versions (WXT, SolidJS, Vitest, …) are pinned by `bun.lock` and
installed automatically; do not install them globally.

Install Bun exactly:

```sh
curl -fsSL https://bun.sh/install | bash -s "bun-v1.3.14"
bun --version    # must print 1.3.14
```

## 1. Clone and install

```sh
git clone https://github.com/edbpede/docrewind.git
cd docrewind

# Install the EXACT pinned dependency graph. --frozen-lockfile fails loudly if
# bun.lock and package.json have drifted, rather than silently resolving new
# versions — this is what makes the build reproducible.
bun install --frozen-lockfile

# Generate WXT's TypeScript types (.wxt/). Wired as `postinstall`, but run it
# explicitly on first setup so tsc and the build can resolve WXT aliases (#imports, @/).
bun run postinstall
```

If you plan to commit, install the git hooks:

```sh
bun run hooks:install
```

## 2. Develop

```sh
bun run dev            # WXT dev server (Chromium) with HMR
bun run dev:firefox    # WXT dev server (Firefox)
```

WXT prints a load instruction; the unpacked dev build lives under `.output/`.
A dev-mode warning that `uno.css` was not found is **expected and harmless** —
UnoCSS styles are applied via the `virtual:uno.css` import at build time.

## 3. Quality gates (run before every PR)

These are the core local quality gates; CI enforces these plus additional workflow checks in `.github/workflows/ci.yml`:

```sh
bun run compile          # tsc --noEmit (strict: noUncheckedIndexedAccess, etc.)
bun run check            # biome check --write . (format + lint + organize imports)
bun run audit:licenses   # reject AGPL-incompatible dependency licenses
bun run test:logic       # Bun — pure decoder/reconstruction/timeline logic
bun run test:coverage    # Bun coverage gate (>=85% on decoder + reconstruction)
bun run test:run         # Vitest — Solid component / storage / browser-API tests
```

> **Test-tier rule.** `bun test` is for **pure, DOM-free logic only**. Solid
> component, storage, and browser-API tests run under **Vitest** with
> `WxtVitest()`. Never run component tests under `bun test`.

## 4. Build and package

```sh
bun run build            # → .output/chrome-mv3  (Chromium MV3 service worker)
bun run build:firefox    # → .output/firefox-mv3 (Firefox MV3 event page)

bun run zip              # → .output/docrewind-<version>-chrome.zip
bun run zip:firefox      # → .output/docrewind-<version>-firefox.zip + -sources.zip
```

The manifest is **generated by WXT** from `wxt.config.ts` — never hand-edit a
`manifest.json`.

## 5. End-to-end & packaging verification

```sh
# E2E runs against a freshly built .output/chrome-mv3 (Chromium only).
bun run build
bunx playwright install --with-deps chromium    # first time only
bun run test:e2e

# Audit the SHIPPED bytes:
bash scripts/verify-manifest.sh             # permission / no-remote-code audit
bash scripts/verify-reproducible-build.sh   # same-environment per-file determinism
bash scripts/checksums.sh                   # → .output/SHA256SUMS
```

**Firefox** has no automated E2E (Playwright can only load a Chromium extension).
Validate it manually with `web-ext` per
[`firefox-validation.md`](firefox-validation.md):

```sh
bun run build:firefox
bun run lint:firefox     # web-ext lint --source-dir .output/firefox-mv3
# then follow the manual checklist in firefox-validation.md
```

## Project layout (where things live)

```
entrypoints/        WXT entrypoints → generated manifest entries
  background.ts        MV3 background (live credentialed retrieval)
  docs.content.tsx     content script: activation affordance on docs.google.com
  replay/              replay tab (page + Web Worker host)
  options/             options / settings page
components/          shared Solid components (PascalCase.tsx)
lib/                 pure + integration modules (kebab-case.ts)
  protocol/            ALL Google Docs protocol assumptions (isolated, PRD §19)
  decoder/             operation-grammar decode (pure; ported grammar, see PRIOR-ART.md)
  reconstruction/      flat character model + apply semantics (pure)
  timeline/            session/timeline derivation (pure)
  domain/              typed domain model + branded IDs
  db.ts, store.ts      IndexedDB (idb) bulk storage + in-memory twin
  settings.ts          WXT typed storage (storage.local, settings only)
  messaging.ts         typed @webext-core/messaging ProtocolMap
e2e/                 Playwright (Chromium) specs + fixtures
scripts/             verification + audit shell/Bun scripts
docs/                this guide, PRD, STACK, RELEASE, protocol-capture, …
```

The decoder / reconstruction / timeline core imports **no** browser/WXT APIs so it
is unit-testable in isolation and storage is swappable in tests (PRD §10.2). All
Google Docs transport assumptions live behind `lib/protocol/*` (PRD §19); the
confirmed protocol facts are recorded in
[`protocol-capture.md`](protocol-capture.md).

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `bun install` reports lockfile drift | You changed `package.json` without updating `bun.lock`. Run `bun install` (no `--frozen-lockfile`) locally, then commit the updated `bun.lock`. |
| `tsc` cannot resolve `#imports` / `@/…` | Run `bun run postinstall` to (re)generate `.wxt/` types. |
| `bun run test:e2e` fails to launch | Run `bunx playwright install --with-deps chromium`; ensure `.output/chrome-mv3` exists (`bun run build`). |
| Solid reactivity warning during dev/build | A React-ism slipped in (destructured props, `.map()` list, effect writing a mirror signal). Fix per `.augment/rules/bun-solid-pro.md`. |
