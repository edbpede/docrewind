# Security Policy

DocRewind is a local-first browser extension with **no backend, no accounts, and
no telemetry** (see [`PRIVACY.md`](PRIVACY.md)). Its attack surface is
correspondingly small: there is no server to compromise and no credential store
to breach. This document records the threat model (PRD §14) and how to report a
vulnerability.

## Reporting a vulnerability

**Please report security issues privately — do not open a public issue for an
unfixed vulnerability.**

- Use GitHub's **[Private vulnerability
  reporting](https://github.com/edbpede/docrewind/security/advisories/new)**
  (Security → Report a vulnerability) on this repository. This is the preferred
  channel and keeps the report confidential until a fix is available.

When reporting, please include:

- a description of the issue and its impact;
- steps to reproduce (a minimal case is ideal);
- affected version / commit and browser; and
- any relevant logs — **with raw document content and revision data redacted**
  (DocRewind itself never logs these; please hold to the same standard).

We will acknowledge your report, work with you on a fix and disclosure timeline,
and credit you in the release notes unless you prefer to remain anonymous. As a
volunteer, AGPL project there is no bug-bounty program.

## Threat model (PRD §14)

### Assets

- Document content and revision history.
- Document metadata and author information.
- Derived writing-process signals (sessions, pauses, large pastes, deletions).

### Risks and mitigations

| Risk | Description | Mitigations |
|------|-------------|-------------|
| **Privacy leakage** (primary) | Accidental leakage via telemetry, logs, diagnostics, exports, browser sync, overly broad permissions, or third-party libraries. | The [privacy guarantees](PRIVACY.md) (PRD §13): no backend/telemetry, local-only storage, never rendering raw bodies/fragments; the anonymized diagnostics model (PRD §10.8); non-synced settings storage; dependency license auditing (`scripts/license-audit.ts`); CI network-isolation audit (runtime + static). |
| **Misinterpreting Google Docs data** (technical) | The revision protocol is undocumented; misreading it could corrupt the reconstruction. | All protocol assumptions isolated in one module (`lib/protocol/*`, PRD §19); fail-safe schema detection that degrades unknown structures to typed opaque placeholders instead of aborting (PRD §9.4); fixture-based decoder/reconstruction tests (PRD §11.5); conservative fidelity claims (PRD §15.3). |
| **Overinterpretation** (user) | A reader treats the replay as proof of misconduct or authorship. | Product positioning ("show what changed", PRD §7); explicit uncertainty disclosure (PRD §9.5, §9.7); reconstruction disclaimers in the UI and docs (PRD §21). |
| **Upstream breakage** (maintenance) | Google changes revision internals, breaking retrieval or decoding. | Protocol isolation (PRD §19) and fixture-driven regression detection; failures surface as clear, typed errors (PRD §10.7) rather than silent corruption. |

### Note on browser sync

Settings are stored in **non-synced** extension storage (`storage.local`), never
`storage.sync`, so preferences and document identifiers cannot leak through a
browser account's cross-device sync.

## Permissions & content security

DocRewind requests the minimum permissions to function:

- `storage` — local persistence only (`storage.local` + IndexedDB).
- `host_permissions: ["*://docs.google.com/*"]` — the sole host; never
  `<all_urls>`.

It declares **no remote code**: no `content_security_policy` whitelisting a remote
origin, no `externally_connectable`, and no third-party `update_url`. The manifest
is generated by WXT and audited on the shipped bytes by
`scripts/verify-manifest.sh`. These properties run in CI on every change.

## Supported versions

DocRewind is pre-1.0. Security fixes are applied to the `main` branch and shipped
in the next release; there is no long-term support branch yet. Always run the
latest release.
